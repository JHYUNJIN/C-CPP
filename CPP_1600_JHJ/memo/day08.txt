프렌드(friend)
	정보를 은폐하면 객체의 신뢰성이 높아지고 기능 개선도 용이하지만
	C++의 액세스 지정자는 매우 엄격하기 때문에 정상적인 문법으로는 외부에서 멤버를 참조할 수 없다.
	하지만, 어떤 경우에는 이런 정보 은폐 기능이 방해가 될 수 도 있기 때문에 
	예외적으로 지정한 대상에 대해서는 모든 멤버를 공개 할수 있는데,
	이를 프렌드 지정이라고 한다.

프렌드의 특징 (프렌드 클래스에 관한 특징)
	- 프렌드 지정은 단방향이며 명시적으로 지정한 대상만 프렌드가 된다.
	
		클래스 A			클래스 B
		friend class B;

		B는 A의 모든 멤버를 액세를 할 수 있다. A는 그렇지 않다.
		자유롭게 사용하고 싶다면 양쪽 모두에 friend를 지정해주면 된다.(양방향 가능)

	- 프렌드 지정은 전이되지 않으며 친구의 친구관계는 인정하지 않는다.
		
		클래스 A			클래스 B			클래스 C
		friend class B;		friend class C;

		A와 C는 friend가 아니다. C는 A에 액세스 할 수 없다.

	- friend class B, C; <-- 이 문법은 허용하지 않는다. 다 따로 지정해주어야 한다.
	- 프렌드 관계는 상속되지 않는다.
